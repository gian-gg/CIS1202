The Yale School of Art website despite its lackluster and chaotic design, serves as an excellent case study for understanding the importance of clean code. HTML Structure

HTML is the backbone of every webpage, defining its content and organization. A well-structured HTML document isn’t just about making things work—it ensures readability, maintainability, and accessibility. By following proper structure, your code becomes easier to understand for both developers and browsers, leading to better performance and user experience.

  • Semantic HTML Elements
  • Proper Nesting & Indentation
  • Overall Structure

Using Semantic Elements

  • Semantic elements like <header>, <article>, and <section> provide meaningful structure to your code. They enhance readability, improve accessibility for screen readers, and help search engines better understand your content.
  • The Yale School of Art website makes use of semantic elements such as <nav>, <header>, <footer>, and <main>. However, despite incorporating these tags, their overall implementation lacks proper organization, affecting clarity and usability.

Proper Nesting & Indentation

  • Properly nested elements and consistent indentation make your HTML more structured, reducing errors and improving collaboration. A well-indented document is easier to debug, maintain, and scale over time.
  • Upon inspecting the Yale School of Art website's source code, it becomes evident that nesting and indentation are inconsistently applied. This results in a cluttered, difficult-to-read structure, making maintenance and debugging more challenging.

Overall Structure

  • A well-organized HTML structure follows a logical hierarchy, making content easy to navigate and maintain. Clearly defined sections contribute to better scalability, accessibility, and overall user experience.
  • In addition to inconsistent nesting and indentation, the Yale School of Art website's source code contains stray elements, such as unclosed or unnecessary tags. These issues not only impact readability but also introduce potential rendering problems, which could be resolved with a more structured approach.

CSS Styling

What is CSS?

CSS (Cascading Style Sheets) is the language used to style and design websites. It controls the visual presentation of HTML elements, including layout, colors, fonts, spacing, and responsiveness. Without CSS, websites would look like basic text documents with no structure or visual appeal.

Why is CSS Styling Important?

CSS is a cornerstone of modern web development for several reasons:

  • CSS makes websites visually engaging, which is crucial for retaining user interest and creating a positive first impression.

  • It ensures a uniform design across all pages, creating a cohesive and professional user experience.

  • CSS allows websites to adapt to different screen sizes, ensuring usability on desktops, tablets, and mobile devices.

  • Proper use of CSS improves readability and usability for all users, including those with disabilities.

Example: Critique of the
Yale School of Art Website

  • Is the layout visually appealing and consistent across the website?

  • Does the website use responsive design techniques? Test it on different screen sizes.

Visual Appeal and Consistency

The website features a high-contrast background image with various colors and elements that overpower the content, making it visually distracting. Different sections utilize inconsistent styles, leading to a fragmented design. Some areas follow a structured layout, while others appear cluttered due to varying font sizes, alignments, and spacing.


A well-structured and visually cohesive website enhances readability and usability. When the design is consistent, users can navigate more efficiently and focus on the content rather than being distracted by visual inconsistencies. Establishing a unified design language helps maintain professionalism and improves user engagement.

Responsive Design

The website integrates responsive design elements such as a dropdown navigation bar for smaller screens and a content structure that shifts dynamically. On mobile devices, side content is repositioned effectively. However, on larger screens, improper alignment and excessive whitespace make the layout appear unbalanced. Some sections remain compact, while others leave large gaps, suggesting an inefficient application of CSS media queries and flex/grid properties.


A fully responsive website ensures accessibility and usability across various devices. Proper layout adjustments improve content presentation, preventing disorganized elements from affecting readability. Implementing a more optimized grid or flexbox structure would allow for better proportional scaling across different screen sizes.

Color, Font, and Spacing

The website employs vibrant and gradient background colors that do not complement the content, reducing readability. Black text over intense backgrounds creates poor contrast, straining the eyes. While the font choice itself is clear and readable, inconsistencies in font sizes and spacing between elements disrupt the overall flow. The right side of the layout contains excessive empty space on larger screens, which could have been utilized for better content distribution.


Effective use of color, font, and spacing significantly enhances user experience. High-contrast backgrounds should be avoided to maintain readability, and consistent typography ensures a smooth visual flow. Addressing spacing issues can create a more structured layout, making the website easier to navigate and visually appealing across all screen sizes.

Usability

Usability pertains to how easily and effectively a user can interact with an application, computer system, or website to accomplish its intended purpose or specific objectives. A highly user-friendly website enhances the user's experience while navigating. It ensures users are aware of their actions and helps them understand where they are within the site.

Organization Problems


As seen in the image above, the website's styling raises concerns, potentially leaving users feeling disoriented and confused during navigation. While the provided links function correctly and are appropriately labeled to indicate their destinations, the overall disorganization of the menus may contribute to user confusion.

Clutters of Confusion


The text in the website is fairly readable, although the line spacing requires some adjustment. However, the design choices raise concerns, specifically the use of different colored backgrounds for each paragraph, coupled with a repetitive background image, may cause users to feel disoriented or confused while reading the content on the website.

Recommendations

  • To enhance user navigation, it is essential to incorporate a well-structured section within the navigation bar. The design should remain straightforward, serving primarily to guide users rather than overwhelm them.


  • The website should be organized into distinct sections, each serving a specific purpose such as a section for content, another for announcements, and a dedicated navigation bar. This approach will mitigate user confusion while navigating the site.


  • Additionally, each text block should feature a cohesive background that complements its section or seamlessly integrates with the overall design. This will help organize the content effectively, facilitating smoother navigation as users scan the website.


  • It is also critical to avoid clutter by not combining all content, navigation elements, announcements, and functionalities into a single area. Such an accumulation would lead to a disorganized and chaotic website, making navigation cumbersome for users.



    • With these practices in mind, the website would significantly enhance user-friendliness for individuals navigating its pages. While there are numerous additional strategies that can further improve the website, I have outlined some fundamental yet essential practices that are crucial for creating a user-friendly experience for all.


    Accessibility

    Ensures that everyone, regardless of ability, can navigate, understand, and interact with a website seamlessly.

    • Problem #1

      The website does not include images, videos, or other multimedia elements that can be accompanied by alt attributes, limiting accessibility for visually impaired users.

      Improvements:

      • Introduce relevant images, videos, or embedded content to enhance the website’s engagement and usability.
      • Ensure that all multimedia elements include descriptive alt text so screen readers can convey their meaning to users with visual impairments.

    • Problem #2

      Although the site supports navigation with the Tab, Shift+Tab, and arrow keys, users struggle to identify which section they are currently on due to low contrast, poor positioning, and lack of visual indicators.

      Improvements:

      • Add clear visual indicators, such as borders or background color changes, to highlight the active section when navigating via keyboard.
      • Optimize the layout by making better use of available space, ensuring that sections are distinguishable and well-structured for easier navigation.

    • An Example of this Navigation Problem:

      The calendar or the subscribe button is currently accessed but is not observable.

    • Problem #3

      The website does not meet proper contrast ratio standards, making it difficult for users with visual impairments to read text and identify interface components effectively.

      Improvements:

      • Adjust color choices to comply with WCAG 2.1 guidelines, maintaining a minimum contrast ratio of 3:1 for UI components and graphical elements.
      • Improve readability by adjusting layout elements, such as centering content and reducing background distractions, ensuring a clearer and more visually accessible experience.
